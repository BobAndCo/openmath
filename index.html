welcome to openmath, a fully open source project to create a math learning resource to everyone

principles

there are already many online resources to learn math, each one specializes in something different yielding pros and cons.

  • youtube - not suitable for offline use, contributing requires computer with video editing software
  • wikipedia contains extraneous information their goal is for general information, not necessarily mathematical information
  • wikibooks contains few complete books, haven't seen active development in a while for mathematical topics, goal is creating books, needs a working knowledge of mediawiki to contribute
  • wikiversitysame as above
  • proofwiki less of a tool to learn topics more of a proof database, relies on mediawiki tech stack
  • nlab higher level mathematics less accessible for beginners, mediawiki
  • brilliant nice entry for beginners lacks cumulative content, payment required
  • textbooks most of them are freely accessible online, but are not easy to navigate (index to page number workflow) and are closed for further development
  • libretexts - quite useful, users cannot easily contribute, complex tech stack
  • proof wiki great site, doesn't focus on the learning aspect just presents proofs, uses recursive learning, but bound within mediawiki's technologies, no external community platform

what openmath tries to do is as follows

  • focus on learning topics in mathematics
  • suitable for offline use
  • contains concise mathematical information
  • simple tech stack (minimal possible dependencies)
  • accessible for all skill levels
  • easy to navigate
  • freely accessible
  • any user can contribute
  • write explanatory proofs
    • when reading a proof you may be able to understand that it's true by following along, then a second analysis is required to figure out how they have come up with the proof, we will try to write explanatory proofs which do both at the same time.
  • keep proofs short containing at most 7 ideas

    it has been shown that our short term memory can on average hold up to 7 items, when writing functions in programming it's useful to write every function in 7 lines so that you can remember every step that your function is doing, then you give it a name which reflects these steps.

additionally it focuses on recursive learning, which is that given a statement for each definition in that statement you can follow it back to a previous definition recursively until you're at some fundamental definition and then work your way back to the original statement. openmath attempts to make recursive learning as smooth as possible

contributing and maintaining

the goal of openmath is not to have a final "edition" done and completed you can imagine it as moving through states where in each state it moves more and more towards being "complete", but is still always be in a continuous state of development, we strive to make sure that at any point openmath is usable for others.

an example to help understand the above paragraph: suppose you might know something is true but don't yet have the proof, if that's the case you may enter the conjecture leave out the proof and leave a warning to other users that it hasn't been proven yet

another user sees this and fills in a proof for the conjecture, now everything holds and we've moved to a more complete state, later on another user simplifies the proof moving the project more towards the complete state

this process continues so that the project moves towards a more complete state

another example could be a user who uploads a image of a proof they did on paper and upload it directly into a proof div, this is an improvement as now there is a proof in place of no proof, in their pull request they mention that they don't know how to use ascii math and this is the best they can do right now

later on a user parses out the latex out of it and into asciimath, and another user grabs any drawings they have and paste them in as images

later on another user sees the pasted images, and uses tikz to generate an svg in place (with included souce code, in case others want to modify it further)

anyone can contribute to openmath, to get started make a github account and then check out the source code

only basic computer usage is required to edit the files, if you see typos or anything you'd like to change then you can edit them in the browser yourself (TODO: how)

when adding theorems the ettiquette is to at a minimum add empty stubs for all of the other theorems, definitions, lemma's corollaries that it uses as dependencies and put (TODO: complete) in them. Doing this will allow other contributors to fill in details or you can come back to them later

additionally you can ask questions and discuss with others using our github discussions, the reason why we use this is that if you're contributing you already have a github account.

basic terminology

  • definition: notation words or anything used to encapsulate something larger into something smaller
  • theorem: something which is true, that takes more effort to show and is most likely used in many places
  • proposition: a theorem of lesser importance
  • lemma: a true statement which is used in proving of a theorem, and not used in many other places
  • corollary: a true statement that doesn't take much effort to prove from a theorem or lemma
  • exercise: something for the reader to try themselves
